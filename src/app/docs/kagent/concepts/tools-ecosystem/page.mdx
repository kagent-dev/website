# Tools Ecosystem

kagent provides a rich set of built-in tools through MCP (Model Context Protocol) servers, plus the ability to add custom tools. Tools are served by two primary MCP servers: `kagent-tool-server` (Kubernetes, Helm, Istio, Cilium, Argo) and the `kagent-grafana-mcp` server (Grafana, Prometheus, Loki, Incidents).

## Configuring Tools on an Agent

### Single MCP Server

```yaml
apiVersion: kagent.dev/v1alpha2
kind: Agent
metadata:
  name: k8s-agent
spec:
  type: Declarative
  declarative:
    modelConfig: default-model-config
    systemMessage: "You manage Kubernetes clusters."
    tools:
      - type: McpServer
        mcpServer:
          apiGroup: kagent.dev
          name: kagent-tool-server
          kind: RemoteMCPServer
          toolNames:
            - k8s_get_resources
            - k8s_describe_resource
            - k8s_get_pod_logs
            - k8s_get_events
```

### With Tool Approval

```yaml
tools:
  - type: McpServer
    mcpServer:
      apiGroup: kagent.dev
      name: kagent-tool-server
      kind: RemoteMCPServer
      toolNames:
        - k8s_get_resources
        - k8s_describe_resource
        - k8s_delete_resource
        - k8s_apply_manifest
      requireApproval:
        - k8s_delete_resource
        - k8s_apply_manifest
```

### Agent as Tool (A2A)

Use another agent as a tool via the A2A protocol:

```yaml
tools:
  - type: Agent
    agent:
      name: specialist-agent
```
